WHAT IS X402?

x402 is the HTTP native payment protocol defined by Coinbase. It uses the HTTP 402 Payment Required status code to enable machine-to-machine and peer-to-peer micropayments. Sling402 implements x402 v1 on Solana — private, instant, zero markup.

HOW X402 PAYMENTS WORK

Step 1: Client requests a paid resource without payment.

Step 2: Server responds with 402 Payment Required and payment requirements:

HTTP/1.1 402 Payment Required

"x402Version": 1
"scheme": "exact"
"network": "solana-mainnet"
"maxAmountRequired": "1000" (micro-USDC, $0.001)
"payTo": "EPgK11Qa7Xectz...Mib2zD"
"asset": "Gh9ZwEmd...tKJr" (USDC mint)
"maxTimeoutSeconds": 300
"resource": "/api/premium-data"

Step 3: Client signs and submits payment on-chain, then retries with proof in the X-PAYMENT header:

X-PAYMENT: base64({
  "x402Version": 1,
  "scheme": "exact",
  "network": "solana-mainnet",
  "payload": { "signature": "5j7s6NiJS3J..." }
})

Step 4: Server verifies the on-chain transaction and returns the resource. All payments are final, instant, and verifiable.

PRIVACY

Sling402 is private by design. The protocol only verifies that a valid payment was made — not who made it. No KYC. No tracking. Payment channels are encrypted end-to-end.
